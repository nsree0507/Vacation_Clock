# Backend Setup - Booking System

## Prerequisites

1. **Node.js** (v14 or higher) - Already installed
2. **MongoDB** - Required for database
   - Local installation: Download from [mongodb.com](https://www.mongodb.com/try/download/community)
   - Or use MongoDB Atlas: [cloud.mongodb.com](https://cloud.mongodb.com)

## Installation & Setup

### 1. Backend Dependencies Already Installed
All required packages are installed. Check `package.json` for the following:
- `express` - Web framework
- `mongoose` - MongoDB ODM
- `cors` - Cross-origin resource sharing
- `dotenv` - Environment variables

### 2. Environment Configuration
The `.env` file is already created with:
```
VITE_API_BASE_URL=https://vacation-clock-6oij.onrender.com/api
PORT=5000
MONGODB_URI=mongodb://localhost:27017/vacation-bookings
NODE_ENV=development
```

### 3. Database Setup

#### Option A: Local MongoDB
```bash
# Windows: Start MongoDB
mongod

# In another terminal, verify connection
mongo
# or use MongoDB Compass GUI
```

#### Option B: MongoDB Atlas (Cloud)
1. Create account at [cloud.mongodb.com](https://cloud.mongodb.com)
2. Create a cluster
3. Copy connection string
4. Update `.env`:
   ```
   MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/vacation-bookings
   ```

## API Endpoints

### 1. Create Booking
**POST** `/api/bookings`

Request body:
```json
{
  "customerName": "John Doe",
  "customerEmail": "john@example.com",
  "customerPhone": "+91 98765 43210",
  "destinationName": "Kerala",
  "packageName": "Adventure Package",
  "travelDate": "2025-06-20",
  "travelers": 2,
  "totalAmount": 35000
}
```

Response:
```json
{
  "success": true,
  "message": "Booking confirmed successfully",
  "booking": {
    "bookingId": "BK-2025-45821",
    "customerName": "John Doe",
    "customerEmail": "john@example.com",
    "customerPhone": "+91 98765 43210",
    "destinationName": "Kerala",
    "packageName": "Adventure Package",
    "travelDate": "2025-06-20",
    "travelers": 2,
    "totalAmount": 35000,
    "bookingStatus": "confirmed",
    "createdAt": "2025-06-18T09:14:18.984Z"
  }
}
```

### 2. Get Booking Details
**GET** `/api/bookings/:bookingId`

Example: `GET /api/bookings/BK-2025-45821`

Response: Same format as booking from POST endpoint

### 3. Get All Bookings (Admin)
**GET** `/api/bookings/admin/all`

Response:
```json
{
  "success": true,
  "count": 10,
  "bookings": [...]
}
```

## Frontend Integration

The frontend automatically:
1. Sends booking data to backend via `BookingPage.jsx`
2. Receives `bookingId` from the backend
3. Displays booking confirmation in `BookingConfirmationPage.jsx`
4. Falls back to temporary booking ID if backend is unavailable

Frontend API client configured in `src/services/api.js`:
```javascript
baseURL: 'https://vacation-clock-6oij.onrender.com/api'
```

## Booking ID Format

Generated automatically by backend:
- Format: `BK-YYYY-XXXXX`
- Example: `BK-2025-45821`
- Year prefix + random 5-digit number

If backend unavailable, temporary ID generated by frontend with same format.

## Troubleshooting

### MongoDB Connection Error
```
MongoDB connection error: connect ECONNREFUSED 127.0.0.1:27017
```
**Solution**: Start MongoDB service or check connection string in `.env`

### Port Already in Use
```
listen EADDRINUSE :::5000
```
**Solution**: Change PORT in `.env` or kill process using port 5000

### CORS Error in Frontend
```
Access to XMLHttpRequest blocked by CORS policy
```
**Solution**: Already configured in backend, ensure `https://vacation-clock-6oij.onrender.com/` is accessible

### Axios Request Timeout
- Frontend automatically falls back to temporary booking ID
- Check if backend server is running (`npm run backend`)
